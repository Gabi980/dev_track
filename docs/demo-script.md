# Demo Script

## Scenario

The presenter demonstrates how a small software team uses DevTrack to manage bugs and tasks.

## Steps

1. Open the application at `http://localhost:5173`.
2. Sign in as Admin with `admin@devtrack.local` and `admin123`.
3. Show the dashboard counters and project sidebar.
4. Create a new project.
5. Create a new bug or task for that project.
6. Assign it to a developer or tester.
7. Move the issue from To Do to In Progress.
8. Open the issue detail panel.
9. Add a comment.
10. Change priority or due date.
11. Use filters and search.
12. Show that activity history and dashboard statistics are updated.

## Expected Result

The system demonstrates authentication, project management, issue tracking, assignment, workflow updates, comments, filtering, dashboard statistics, and traceability.
